阅读规范

设计你的API

设计API很容易。但是设计一个长期存在且用户喜欢的API - 这有点难度。但是使用RAML，您可以充分利用完整的API设计生命周期，这意味着您可以直观地设计API，测试它并获得用户反馈，而无需编写任何代码。

视觉设计

RAML允许您以人类可读的格式描述您的API - 换句话说，纯文本。当您使用常识术语定义RESTful API时，使用API​​ Workbench或API Designer等工具，您可以直观地看到API的样子。这意味着对于每个按键，您都可以直观地了解API的外观，感觉和功能。这也意味着RAML易于学习和易于使用 - 即使您不熟悉设计RESTful API。

代码重用

除了能够以您希望的方式直观地验证您的API外观，感觉和功能之外，RAML还允许您合并两个开发人员最佳实践：设计模式和代码重用。RAML不是必须依靠人工验证来确保您的API一致，而是让您利用库，覆盖，特征和资源类型来确保您的API始终保持一致 - 并在此过程中为您节省大量的返工。

原型您的REST API

认为您的API满足用户的需求是不够的 - 您应该知道它确实如此。使用RAML，您可以轻松地对API进行原型设计，并将其发送给全球潜在的API用户或合作伙伴。这意味着您可以随时随地与任何人共享API模拟。

加上RAML使这些开发人员可以轻松地试用您的API，并在流行的API工具（如Postman）中内置集成。您甚至可以使用API​​ Notebook通过模拟API引导用户，或者更好的是，让他们为您提供他们如何与您的API进行交互的真实用例 - 再次无需任何人编写一行代码！

完善您的REST API

因为您能够设计，原型化并获得有关RESTful API设计的反馈，所以您可以在生产之前消除设计缺陷和缺陷。这意味着您可以利用API设计周期，并创建一个您知道用户会喜欢的API，以及经过深思熟虑并经过仔细测试以确保其足够长久的API。

强大的工具

RAML在首次推出时完全改变了API行业，将重点从“记录您的API”转移到包含完整的API生命周期。RAML的生态系统是让开发人员能够做更多的事情，而且随着RAML的发展，工具也是如此。

API Workbench

API Workbench是第一个功能齐全的API设计IDE。API设计不再是二等公民，但现在开发人员可以在任何地方使用他们的API设计，同时还可以利用内置的Git支持，内嵌控制台，轮廓和重构工具，轻量化的实用程序。
