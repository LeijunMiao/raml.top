以简单方法设计API

RESTful API建模语言（RAML）可以轻松管理从设计到共享的整个API生命周期。它简洁 - 您只需编写您需要定义的内容 - 并且可以重复使用。它是机器可读的API设计，实际上是人类友好的。

写一次。使用很多。鼓励创造性的懒惰。

使用RAML设计您的API

RAML可让您在设计时看到API的外观，使用易于阅读的纯文本。无需编写单行代码，您不仅可以完善您的API设计，还可以为客户，合作伙伴甚至您自己的内部工程师创建功能齐全的模拟器，以便进行审查和构建。

使用RAML构建您的API

RAML使开发人员和工程师能够做他们最擅长的事情 - 编写代码。没有更多的猜测或获得API的中途只是为了实现设计不起作用或不一致。您还可以使用许多可用的工具来帮助您从RAML文件生成API的一般结构，从而缩短产品上市时间（以及基于错误的麻烦）。

使用RAML测试您的API

大多数工程师讨厌编写单元测试。RAML允许您通过运行简单的shell命令来生成大多数测试。不再需要手动测试（虽然我们当然建议您自己创建），甚至更好 - 您甚至可以使用API​​ Fortress，API Science，Parasoft，Postman和SmartBear等第三方服务来满足您的测试需求。

使用RAML记录您的REST API

告别花费数小时编写和维护API文档。使用RAML，您可以轻松生成所有文档，只需单击按钮即可提供最新文档。
