阅读规范

分享和支持您的 API

RAML的优点在于它不仅可以让您设计API，还可以实际定义您的API。这意味着可以使用这个相同的RAML规范将您的API与其他系统集成，甚至可以为当今最流行的语言生成SDK。

此外，使用 API Notebook 您甚至可以为您的API提供交互式演练和示例用例，同时还允许开发人员以标准格式与您的支持团队进行通信 - 使他们的工作变得非常简单！

SDK 生成

在Java，.NET，PHP，Ruby，NodeJs，iOS，Windows，Go等中为RESTful API创建代码库就像单击按钮一样简单！

您可以找到几种不同的社区工具来创建这些SDK，以便您的工程师不必这样做！

您还可以利用 APIMatic.io 和 RESTUnited, 等专业服务，它们提供三种SDK及其免费计划，并支持所有上述语言等。

这意味着每次更新API时都可以轻松创建更新的代码库，而不需要工程团队的一部分 - 让他们将时间集中在更重要的事情上 - 比如新功能！

第三方工具

正如您可以使用RAML创建SDK一样，许多公司都有内置支持，可让您自动获取API的资源，方法和属性！

例如，Oracle和MuleSoft提供的企业工具可以让您轻松连接到RAML中描述的任何API，只需粘贴RAML规范URL即可。不再需要猜测，也不再需要在这些流行的工具中手动设置API调用。

API 笔记本

API Notebook彻底改变了开发人员与API交互的方式，让他们可以轻松地尝试进行调用，操作数据，甚至使用基本JavaScript同时连接到多个API！

使用简单的降价功能，您可以创建无限的笔记本，演示不同的用例或如何使用API​​的示例。

想了解更多？只需查看其中一些示例用例（并在apinotebook.com上了解更多信息）：

  • 在Instagram上搜索可爱的小猫
  • 在Twitter上找到你最后的追随者
  • 获取GitHub列表的列表

正如您所看到的，API Notebook可以完成所有猜测，开发人员可以使用您的API来完成最受欢迎或最重要的用例。

处理支持

但是，API笔记本真正独一无二的是任何人都可以创建自己的笔记本。用户可以轻松克隆现有笔记本或创建新笔记本以分享其确切的用例 - 让您立即看到他们真正拥有的问题。

这意味着不再需要通过专有的部分代码进行挖掘，以便弄清楚bug是在他们的代码中还是在你的API中 - 而是立即知道他们在调用API的过程中是否做错了什么，或者是否存在错误你的角色。

这也可以让您的用户调试他们自己的调用，以确保他们正确的做事！

RAML是唯一一个旨在以人类可读格式包含完整API生命周期的规范，其中包含代码和设计模式重用。使用RAML，您可以使用一个规范真正 设计, 构建, 测试, 记录, 和 共享 您的API。
