测试你的API

轻松测试

单元测试对于确保您的API没有任何向后兼容性中断并满足您当前的所有要求绝对至关重要。使用RAML，您可以以这样的方式描述您的HTTP REST API，从而可以为您生成许多测试！

这意味着在定义规范后，您可以利用TDD（测试驱动开发）并在CI（持续集成）环境中自动生成测试。

流行的测试工具

您可以使用几种工具来生成RAML测试，包括许多社区免费提供的测试，以及通过API Science和SmartBear等公司提供的其他更广泛的测试服务。

Abao

Abao是一个NodeJS命令行工具，用于测试以RAML格式编写的API文档，以及后端实现。使用Abao，您可以轻松地将API文档插入到Travis CI或Jenkins等持续集成系统中，并始终保持API文档的最新状态。Abao使用Mocha（http://mochajs.org/）来判断特定API响应是否有效或是否有效。

更多信息下载


Vigia

Vigia是一个适应性强的API集成测试套件，支持基于RAML定义文件的测试生成。

更多信息下载


Postman

Postman是当今开发人员使用的最流行的API调用和测试工具之一。Postman作为Chrome应用程序免费提供，支持对任何RESTful API的API调用，并允许您在导入RAML规范后设置脚本和测试。

更多信息下载


热门测试服务

API Fortress

API Fortress通过检查API中的延迟和响应速度来提供测试。使用API​​ Fortress，您还可以验证响应和有效负载，以确保在开发或生产中您的API是否正常运行。最重要的是，他们的服务，API Fortress提供自己的API - 让您按需测试您的API。

访问网站


API Science

API Science提供全球监控和测试API，以识别性能问题，中断，错误。使用API​​ Science，您可以测试基于HTTP的REST API的多个方面，包括JSON，OAuth和XML。您甚至可以在生产中测试真实的高级CRUD序列，并通过Slack，PagerDuty或Web钩子接收警报。

访问网站


SmartBear

SmartBear为您的API提供了大量测试工具，让您可以提取RAML规范来识别延迟/速度问题，错误并验证响应数据。除API准备工具外，它们还提供API虚拟化，持续集成工具和性能测试。
